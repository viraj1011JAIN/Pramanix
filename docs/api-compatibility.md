# API Compatibility and Semver Contract

Pramanix follows [Semantic Versioning 2.0.0](https://semver.org/). This document defines the exact public API surface, what constitutes a breaking change, and how contributors must update the contract test snapshots when making intentional changes.

## 1. What is public API

The stable public surface is the union of all names in `pramanix.__all__` as locked in `tests/unit/test_api_contract.py`.

| Surface | Stability guarantee |
|---|---|
| `pramanix.__all__` names | Stable; removals/renames require MAJOR bump |
| `SolverStatus` member names and `.value` strings | Stable; values appear in persistent audit logs |
| `SolverStatus` iteration order | Stable; reordering requires MINOR bump |
| `Decision.to_dict()` key set | Stable; this is the wire schema |
| `Decision.to_dict()` per-field types | Stable; type changes break deserialisation code |
| `Decision` factory methods (`safe`, `unsafe`, `error`, `timeout`, `stale_state`, `validation_failure`) | Stable |
| `GuardConfig` field names | Stable; removals/renames require MAJOR bump |
| `GuardConfig` default values (locked set) | Stable; changes may alter behaviour for existing deployments |
| `Guard.verify()` / `verify_async()` / `parse_and_verify()` signatures | Stable |
| Exception class hierarchy under `PramanixError` | Stable |
| Environment variable names (`PRAMANIX_*`) | Stable |

## 2. What is NOT public API

The following are **internal implementation details** and may change in any release without a version bump:

- Any module not listed in `pramanix.__all__` (e.g. `pramanix.solver`, `pramanix.fast_path`, `pramanix.transpiler`)
- Any name prefixed with `_` (e.g. `_BLOCKED_STATUSES`, `_fast_path_hit`, `_DecisionBuilder`)
- Internal dataclass fields not accessible through the public constructor (e.g. `Decision._raw_z3_model`)
- The exact Z3 SMT formula generated by a policy
- The internal worker process communication protocol
- The exact wording of `explanation` strings in `Decision.to_dict()` — these are human-readable and may improve across patch releases
- Test fixtures, conftest helpers, and benchmark scripts
- `htmlcov/` and other build artefacts
- The exact log format emitted to `stderr` / structured logging sinks

## 3. Semver rules

### MAJOR bump (breaking)

- Remove or rename any name from `pramanix.__all__`
- Remove or rename a `SolverStatus` member
- Change the `.value` string of an existing `SolverStatus` member (corrupts stored audit logs)
- Remove a key from `Decision.to_dict()`
- Change the Python type of an existing `Decision.to_dict()` value (e.g. `str` → `bytes`, `list` → `tuple`)
- Remove or rename a `GuardConfig` field
- Change the behaviour of an existing `GuardConfig` field such that a previously-valid config raises an error
- Change a default value in a way that makes existing deployments less safe or less correct
- Remove a `Guard` method from the public interface
- Change a positional argument to `Guard.verify()` / `verify_async()` / `parse_and_verify()`
- Make `GuardConfig` mutable
- Remove a `Decision` factory method

### MINOR bump (additive, non-breaking)

- Add a new name to `pramanix.__all__`
- Add a new `SolverStatus` member (must also classify it in `_BLOCKED_STATUSES` or as OBSERVABILITY)
- Add a new key to `Decision.to_dict()`
- Add a new `GuardConfig` field with a default value
- Add a new optional keyword argument to a `Guard` method
- Add a new exception class under `PramanixError`
- Add a new `PRAMANIX_*` environment variable with a sensible default
- Reorder `SolverStatus` members (observable only in `for m in SolverStatus` loops)
- Tighten a `GuardConfig` default value in a way that is strictly safer

### PATCH bump (fix, non-breaking)

- Bug fixes that do not change observable behaviour
- Documentation corrections
- Performance improvements with no API surface change
- Dependency version bumps (patch-compatible)
- Improve `explanation` wording in `Decision` factories

## 4. Serialisation compatibility — Decision.to_dict()

`Decision.to_dict()` is the canonical wire format. It is used by:

- The audit log subsystem (written to persistent storage, may survive multiple major releases)
- The CLI command `pramanix audit verify` (reads stored JSON records)
- Any downstream consumer that deserialises a `Decision` from JSON

### Locked wire schema (v0.9.0)

| Key | Type | Notes |
|---|---|---|
| `decision_id` | `str` (UUID4) | Distributed trace key; must remain UUID4 |
| `allowed` | `bool` | Exactly `bool`, not `int` |
| `status` | `str` | Plain `.value` string, never a `SolverStatus` instance |
| `violated_invariants` | `list[str]` | Empty list for SAFE decisions |
| `explanation` | `str` | Human-readable; wording may change in patches |
| `solver_time_ms` | `int \| float` | Must be ≥ 0 |
| `metadata` | `dict` | Caller-supplied; content not locked |
| `intent_dump` | `dict` | Runtime intent data; content not locked |
| `state_dump` | `dict` | Runtime state data; content not locked |
| `decision_hash` | `str` | 64 lowercase hex chars (SHA-256); must be deterministic |
| `signature` | `str \| None` | Base64/hex encoded; **must not be `bytes`** — bytes breaks JSON |
| `public_key_id` | `str \| None` | None when unsigned |
| `policy_hash` | `str \| None` | None when no expected hash configured |

### Serialisation guarantees

- `decision_hash` is deterministic: the same `Decision` instance produces the same hash on every `to_dict()` call.
- `status` is always a plain `str` (the `.value`), never a `SolverStatus` enum instance. This ensures compatibility with stdlib `json.dumps` without a custom encoder.
- `signature` is always `str | None`. A future signing implementation must encode binary signatures to hex or base64 before writing to `to_dict()`.
- Pramanix attempts `orjson` serialisation where available; the fallback is stdlib `json`. The wire format is identical in both paths.

### orjson / stdlib fallback

If `orjson` is installed, Pramanix uses it for ~3× faster serialisation. The output is byte-for-byte equivalent for all types that appear in `Decision.to_dict()`. If you add a new field whose type is not supported by both serialisers, it is a MINOR change and you must document the type constraint here.

## 5. Enum stability — SolverStatus

`SolverStatus` is a `StrEnum`. Its stability requirements are more precise than a plain class.

### Member names (MAJOR to change)

Every `.name` string (e.g. `"SAFE"`, `"UNSAFE"`) is a public identifier. Code that does `SolverStatus["SAFE"]` or `match decision.status: case SolverStatus.SAFE:` depends on these names.

### Wire values (MAJOR to change)

Every `.value` string (e.g. `"safe"`, `"unsafe"`) is written to persistent audit records. **Changing a wire value is a data-corruption event for any existing audit store.** This requires a major version bump AND a migration guide in `CHANGELOG.md`.

### Iteration order (MINOR to reorder)

Python enums preserve insertion order. `list(SolverStatus)` currently produces members in this exact sequence:

```
SAFE → UNSAFE → TIMEOUT → ERROR → STALE_STATE →
VALIDATION_FAILURE → RATE_LIMITED → CONSENSUS_FAILURE → CACHE_HIT
```

Reordering members changes the output of `for m in SolverStatus` loops and may affect documentation that lists members in source order. This is treated as a MINOR change.

### StrEnum identity contract

Because `SolverStatus` inherits from `StrEnum`, every member satisfies `member == member.value`. Callers may compare a status with a plain string (e.g. `decision.status == "safe"`). This contract must be preserved when adding members.

### Adding members (MINOR)

A new member must be classified immediately in `decision.py`:

- Add to `_BLOCKED_STATUSES` if it means the request was denied.
- Document as OBSERVABILITY-only if it does not block (like `CACHE_HIT`).
- Never leave a new member unclassified — the contract test `test_safe_is_sole_non_blocked_non_observability` will fail and block CI.

### Removing members (MAJOR)

Callers may hold references to a `SolverStatus` value (e.g. stored in a database, pattern-matched in application code). Removing a member is always MAJOR.

## 6. GuardConfig compatibility

`GuardConfig` is a frozen dataclass. All fields have default values so `GuardConfig()` with no arguments is always valid.

### Stable defaults

The following defaults are part of the public contract. Changing them without a version bump silently alters the behaviour of every deployment that relies on the default:

| Field | Default | Notes |
|---|---|---|
| `execution_mode` | `"sync"` | Controls worker dispatch strategy |
| `solver_timeout_ms` | `5_000` | (*) Z3 solver timeout |
| `max_workers` | `4` | Worker pool size |
| `max_decisions_per_worker` | `10_000` | Worker recycle threshold |
| `worker_warmup` | `True` | Pre-warm worker processes |
| `log_level` | `"INFO"` | Structured log verbosity |
| `metrics_enabled` | `False` | Prometheus metrics |
| `otel_enabled` | `False` | OpenTelemetry tracing |
| `translator_enabled` | `False` | LLM intent translator |
| `fast_path_enabled` | `False` | Rule-based fast path |
| `fast_path_rules` | `()` | Fast path rule list |
| `signer` | `None` | Caller-injected; not set by default |
| `solver_rlimit` | `10_000_000` | (*) Z3 anti-DoS resource limit |
| `max_input_bytes` | `65_536` | (*) 64 KiB input cap |
| `min_response_ms` | `0.0` | Minimum artificial response delay |
| `redact_violations` | `False` | Redact violated invariant names |
| `expected_policy_hash` | `None` | Deployment-specific; caller sets this |
| `injection_threshold` | `0.5` | (*) Injection detection confidence gate |

Fields marked (*) encode a **security or safety guarantee**. Changing them requires an explicit security review comment in the PR.

### Intentionally evolvable defaults

The following fields may have their defaults tightened across patch releases without a version bump, because any change is strictly safer:

| Field | Rationale |
|---|---|
| `shed_latency_threshold_ms` | Adaptive shedder tuning — may be reduced as benchmarks improve |
| `shed_worker_pct` | Adaptive shedder tuning — may be reduced as benchmarks improve |

### Immutability requirement

`GuardConfig` must remain `frozen=True`. Workers receive a `GuardConfig` reference; mutability would create race conditions. Making `GuardConfig` mutable is a MAJOR breaking change.

### Zero-argument constructor guarantee

`GuardConfig()` with no arguments must always succeed. Adding a required field (one without a default) is a MAJOR breaking change.

### Valid execution modes

The currently accepted values for `execution_mode` are: `"sync"`, `"async-thread"`, `"async-process"`. Any other value raises `ConfigurationError`. Adding a new valid mode is MINOR; removing an existing valid mode is MAJOR.

## 7. Deprecation policy

Before removing a public name or changing behaviour in a breaking way:

1. Mark it deprecated for **at least one minor release** using a `DeprecationWarning` at the call site.
2. Document the replacement in `CHANGELOG.md` under the MINOR release that added the deprecation.
3. Remove in the next **major** release.
4. Update `tests/unit/test_api_contract.py` when the removal lands.

This gives downstream users at least one release cycle to migrate before the name disappears.

For `SolverStatus` wire values specifically: because changing a value corrupts audit logs, deprecation is not sufficient — a migration guide with a data-migration script must be provided alongside the MAJOR bump.

## 8. How to update contract tests

When you deliberately change the public API surface (adding, removing, or modifying something covered by the contract tests), follow these steps exactly:

### Step 1 — Confirm the semver category

Consult [section 3](#3-semver-rules). Every change falls into MAJOR, MINOR, or PATCH.

### Step 2 — Update the snapshot in test_api_contract.py

Open `tests/unit/test_api_contract.py` and find the relevant snapshot constant:

| What changed | Constant to update |
|---|---|
| `pramanix.__all__` | `_EXPECTED_ALL` |
| `SolverStatus` members or values | `_EXPECTED_SOLVER_STATUS_ORDERED` |
| `Decision.to_dict()` keys | `_EXPECTED_DECISION_KEYS` |
| `Decision.to_dict()` field types | The corresponding `test_<field>_is_*` method |
| `GuardConfig` field names | `_EXPECTED_GUARDCONFIG_FIELDS` |
| `GuardConfig` default values | `_EXPECTED_GUARDCONFIG_DEFAULTS` |

Update **only the snapshot you are explicitly changing**. Do not refactor unrelated tests.

### Step 3 — Write a CHANGELOG.md entry

Add an entry under `## [Unreleased]` in `CHANGELOG.md`:

```markdown
### <Added|Changed|Removed|Fixed>
- <Brief description of the API change and why it was made.>
  Semver category: MAJOR / MINOR / PATCH.
  Contract test updated: `<ConstantName>` in `tests/unit/test_api_contract.py`.
```

### Step 4 — Bump the version if MAJOR

If the change is MAJOR:

1. Update the `version` field in `pyproject.toml`.
2. Update `__version__` in `src/pramanix/__init__.py`.
3. Both values must match exactly.

### Step 5 — Run contract tests

```bash
pytest tests/unit/test_api_contract.py -v
```

All tests must pass before the PR is merged. A failing contract test that you did not update is a blocking review comment.

### Step 6 — For SolverStatus wire value changes only

Provide a data migration script in `docs/migrations/` that shows users how to update existing audit log records. Link to it in the CHANGELOG entry.

These tests run in CI on every PR. A failure means the public API changed without an explicit snapshot update.

| Test class | What it locks |
|---|---|

## Contract enforcement

The following tests will fail immediately if the contract is violated:

| Test class | What it locks |
|---|---|
| `TestAllExportsLock` | `pramanix.__all__` exact set |
| `TestSolverStatusLock` | `SolverStatus` members and wire values |
| `TestDecisionToDictLock` | `Decision.to_dict()` key set and value types |
| `TestGuardConfigFieldLock` | `GuardConfig` field names and immutability |

When one of these tests fails due to a deliberate API change:

1. Update the snapshot constant in `tests/unit/test_api_contract.py`.
2. Determine the semver category (major/minor/patch) from the table above.
3. Update `CHANGELOG.md` under the appropriate `## [Unreleased]` section.
4. If major: bump the major version in `pyproject.toml` and `src/pramanix/__init__.py`.
